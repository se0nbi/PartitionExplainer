# PARSE: Partition-based Shapley–Taylor Explanation for Graph Neural Networks

PARSE is a post-hoc explanation framework for Graph Neural Networks. Instead of scoring nodes or
edges in isolation, it decomposes a GNN's class logit — relative to a fully masked baseline — into the
**intrinsic contributions of disjoint connected subgraphs** and the **pairwise interactions between
them**, so the explanation captures how substructures cooperate to drive a prediction.

The decomposition combines **order-2 Shapley–Taylor interaction indices** with **Myerson connectivity
constraints**. Node contributions are estimated once by Monte-Carlo permutation sampling and cached in
an interaction matrix; an interpretable partition is then found by simulated annealing without further
GNN queries. The scoring objective (entropy/size/coverage weights, the group-size law, the
interaction-augmentation terms, and a structure-aware propagation strength) is **calibrated per dataset**
by a surrogate optimizer with a Pareto-guarded accept gate that adopts a configuration only when it
improves held-out faithfulness.

Across fifteen datasets (six synthetic, nine real-world) and ten competing explainers, PARSE attains the
best faithfulness on **51 of 60** (dataset, metric) comparisons and is the top method on every dataset.
See [`RESULTS.md`](RESULTS.md) for the full leaderboard.

## Repository layout

| Path | Description |
|---|---|
| `src/` | All Python modules (detailed below). |
| `notebook.ipynb` | Core framework, controlled AND/OR/DIST interaction tasks, and the public-benchmark pipeline. |
| `functionalgroup.ipynb` | Molecular case studies (Tox21 SR-MMP); trains the model and renders the case-study figures. |
| `hpo_tuning.ipynb` | Visualizes the learned per-dataset calibration (generated by `src/reporting/_make_hpo_nb.py`). |
| `figures/` | Generated paper figures (leaderboard + molecular case studies). |
| `outputs/hpo/eval_records/` | Per-graph faithfulness records: `main/` (PARSE + native baselines, one file per dataset), `baselines/<method>/` (one folder per expensive baseline), `gsiq25/` (GraphSHAP-IQ). |
| `outputs/hpo/hyperparameters/` | Calibrated configs: `best/` (per-dataset HPs), `state/` (search traces), `calibration/` (surrogate + cross-dataset artifacts). |
| `outputs/hpo/board/` | `overall_winners_FINAL16.csv` (leaderboard) and `report.csv` (default → tuned). |
| `outputs/models/`, `outputs/case_studies/`, `outputs/dataset_stats.csv` | Trained Tox21 checkpoint + metrics; case-study tables; dataset statistics. |

Inside `src/` (run scripts from the repository root, e.g. `python src/evaluation/_paper_artifacts.py board`):

| Folder | Modules |
|---|---|
| `core/` | `_hpo_engine.py` (calibration engine + framework harness), `_board_core.py` (leaderboard scoring), `_hpo_run.py` (search driver) |
| `evaluation/` | `_paper_artifacts.py` (tables + comparison figure), `_ovn.py` (structure-aware board tuning) |

| `reporting/` | `_regen_casestudy.py` (molecular case-study figures), `_make_hpo_nb.py` (generates `hpo_tuning.ipynb`) |

## Installation

```bash
pip install -r requirements.txt
```

PyTorch Geometric and its companion packages (`torch_scatter`, `torch_sparse`, `torch_cluster`,
`pyg_lib`) must match your installed PyTorch / CUDA build — install them from the official PyG wheel
index. RDKit and `ugropy` are only needed for the molecular case studies.

## Datasets

Most benchmarks (BA-2Motifs, Mutagenicity, PROTEINS, BACE, BBBP, MUTAG, Benzene, ENZYMES, Tox21) are
downloaded automatically on first use via [DIG](https://github.com/divelab/DIG) into `outputs/dig_data/`.
Graph-SST2 and Graph-SST5 are large and must be placed under `outputs/dig_data/Graph-SST2/` and
`outputs/dig_data/Graph-SST5/` separately.

## Baselines

The third-party baselines (MAGE, GStarX, GraphXAI, INSIDE-SHAP) are not vendored here. Clone them into
`_vendor/` to enable the corresponding wrappers; GraphSHAP-IQ is installed via `pip install shapiq`.

## Reproducing the results

Run everything from the repository root. The evaluation records under `outputs/hpo/` let you
regenerate the paper tables and figure directly:

```bash
python src/evaluation/_paper_artifacts.py board   # leaderboard -> outputs/hpo/board/overall_winners_FINAL16.csv
python src/evaluation/_paper_artifacts.py fig     # comparison figure -> figures/fig_leaderboard.pdf
python src/evaluation/_paper_artifacts.py stats   # dataset statistics -> outputs/dataset_stats.csv
```

Other entry points:

```bash
python src/reporting/_regen_casestudy.py                       # molecular case-study figures
python src/core/_hpo_run.py --datasets bace bbbp ...      # re-run per-dataset calibration
```

The controlled interaction tasks (correctness + counterfactual validation) and the public-benchmark
evaluation run from `notebook.ipynb`; the molecular case studies run from `functionalgroup.ipynb`.
